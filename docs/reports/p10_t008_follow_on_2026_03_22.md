# P10-T008 Follow-On: Runtime to Pipeline Registry Handoff

Date: 2026-03-22

## Goal

Make `execution/flagship_pipeline.py` and `training/runtime/experiment_registry.py` consume the executable multimodal runtime result, not the plan-only training contract.

## Exact Checkpoints To Preserve

The next handoff should keep these identities aligned end to end:

- `training_result.checkpoint.checkpoint_tag`
- `training_result.checkpoint.checkpoint_ref`
- `training_result.checkpoint.run_id`
- `training_result.state.state_signature`
- `training_result.state.dataset_signature`
- `experiment_registry.records[0].checkpoint_tag`
- `experiment_registry.records[0].checkpoint_ref`
- `experiment_registry.records[0].experiment_id`
- `flagship_pipeline.pipeline_signature`

## What Must Change

1. `training/runtime/experiment_registry.py` should build registry records from the executable runtime result, not from `prepare_multimodal_training`.
2. `execution/flagship_pipeline.py` should consume the runtime checkpoint and registry identity that come out of the executable path.
3. The pipeline should continue to surface blockers honestly if the runtime is still partial, but it should no longer be anchored to the planner-only contract surface.

## Likely Regression Traps

- `experiment_id` still hashes plan-only fingerprints instead of runtime checkpoint identity.
- `checkpoint_ref` in the registry diverges from the runtime checkpoint ref.
- `pipeline_signature` changes because the registry surface changes, but the tests still expect the old plan-only hash.
- Pipeline status becomes too optimistic when the runtime is only partially executable.
- The integration test still exercises the old contract shape and misses the runtime checkpoint round-trip.

## Focused Validation

```powershell
python -m pytest tests\unit\training\test_experiment_registry.py tests\integration\test_flagship_pipeline.py tests\unit\training\test_multimodal_runtime.py
python -m ruff check execution\flagship_pipeline.py training\runtime\experiment_registry.py training\multimodal\runtime.py tests\unit\training\test_experiment_registry.py tests\integration\test_flagship_pipeline.py tests\unit\training\test_multimodal_runtime.py
```

## Success Bar

P10-T008 is done when the pipeline and registry both point at the same runtime checkpoint identity, the pipeline remains honest about any remaining runtime limits, and the tests prove the runtime result survives through registry and pipeline serialization without falling back to the planner-only contract.
